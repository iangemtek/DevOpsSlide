DevOps

2015/06/17

What is DevOps?

A new term that primarily focuses on improving collaboration, communication and integration between software developer and IT operations.

From Waterfall to Agile

Waterfall

  • 100% of the system is defined before development
  • Inflexible and monolithic

Agile

  • Improved collaboration between business users and developers
  • Software evolves iteratively
  • Continuous improvement and feedback

Agile

From Agile to DevOps

DevOps

  • Agile development move downstream towards infrastructure
  • Improved collaboration between developers and IT operators
  • More reliable and efficient in deploying applications

DevOps

DevOps with infrastructure

  • provisioning

Principals

  • infrastructure as code
  • continuous deployment
  • automation
  • monitoring
  • security

Infrastructure as code

  • Code has version management
  • Same idea applies to infrastructure provisioning
  • Define configuration in a declarative way
  • Configuration stored in a version management system
  • "Infrastructure code" used in provisioning, orchestration, and deployment

Continuous Deployment

  • Automated deplayment of production-ready application code
  • Deploy code to different environment
  • Automatically rolled back on failure

Automation

  • From setup, configuration to deployment and support
  • For application and infrastructure
  • Eliminate manual processes

Monitoring

  • Feedback help improve the communication and collaboration
  • Monitoring, alerting and Auditing infrastructure
  • ... and application (?)

Continuous Deployment

Automation
